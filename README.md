# Модуль ИИ-анализа аномалий в биометрических данных

## Описание проекта

**Модуль ИИ-анализа аномалий** в биометрических данных, получаемых с камер видеонаблюдения — система для выявления несоответствий между признаками (рост, пол, тип фигуры) и снижения количества ложных срабатываний в системах безопасности **на 30–40%**.

---

## Проблема

Системы видеонаблюдения часто ошибаются при распознавании людей из-за:

- Плохого освещения
- Маскировки
- Технических сбоев
- Неточности детекторов

**Последствия:** ложные тревоги, перегрузка операторов, потеря доверия к системе.

---

## Решение

Мы создаем систему обнаружения аномалий, которая:

### 1. Получает данные от детектора силуэта

**Входные данные:**
- Последовательность кадров с координатами силуэта человека
- Высота, ширина, скорость движения, направление
- Потоковое видео в реальном времени

**Технологии:** готовый детектор (YOLOv8 + OpenPose) или симуляция

### 2. Оценивает логическую согласованность признаков

**Анализируемые параметры:**
- Соотношение рост/ширина плеч
- Скорость походки vs. тип фигуры
- Динамика движений (неестественные остановки, резкие изменения направления)

**Эталонные профили:**
| Группа | Рост | Ширина плеч | Особенности |
|--------|------|-------------|-------------|
| **Мужчины** | 175–185 см | 45–55 см | - |
| **Женщины** | 160–170 см | 35–45 см | - |
| **Дети** | < 150 см | - | Энергичная походка |

### 3. Выявляет аномалии и классифицирует их причины

**Типы аномалий:**
- **Ошибка детектора** (некорректное определение пола)
- **Маскировка** (широкая одежда, искажающая параметры)
- **Технический сбой** (резкие изменения между кадрами)

**Формат вывода:**
> *«Аномалия: вероятность маскировки — 89%. Причина: ширина плеч 60 см при росте 170 см — не соответствует ни одному типу фигуры»*

### 4. Интегрируется в систему видеонаблюдения

**Функциональность:**
- Работа в реальном времени
- Сигнализация при обнаружении аномалий (звук, уведомления, логи)
- REST API для интеграции с внешними системами

### 5. Обучается на своих ошибках

**Механизм обучения:**
- Разметка ложных срабатываний оператором
- Online learning для дообучения модели
- Сохранение истории аномалий для анализа и отчетности

---

## Результат

**Интеллектуальный аудитор**, который проверяет логику системы и защищает её от ошибок — человеческих и технических.
